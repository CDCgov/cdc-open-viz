# AI Adoption

## We are using AI for...
  - Large Feature Enhancements: E.g. Creating New Visualizations - Guage Chart, Heat Map, etc.
  - Small/Medium Feature Enhancements: build a chart/editor/config change across multiple packages while following existing repo patterns.
  - Code review: review a branch or diff for regressions, missing tests, migration risks, and edge cases. Branch Auto Commenting Rules, Local Review, etc.
  - Bug triage and fixes: trace a rendering or config bug, identify the code path, patch it, and run targeted tests. Close Github Issues, etc.
  - Test generation: add or repair unit tests and Storybook tests for visualization/editor behavior.
  - Migration work: update shared config migrations, version handling, and legacy compatibility logic while checking the migration rules docs first.
  - Config maintenance: when consumer-facing config changes, update the relevant CONFIG.md docs in the same pass.
  - Refactors: rename APIs, move shared helpers, tighten package boundaries, and clean up deprecated paths.
  - Repo documentation: maintain READMEs, developer docs, and internal context docs for complex systems.
  - Branch analysis: compare a feature branch against dev and summarize what changed.
  - Test execution workflow: playwright, backstop, jest, etc.
