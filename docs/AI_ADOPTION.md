# AI Adoption

## We are using AI for...
- [x] Large feature enhancements: for example, creating new visualizations such as gauge charts and heat maps.
- [x] Small and medium feature enhancements: building chart, editor, or config changes across multiple packages while following existing repo patterns.
- [x] Code review: reviewing a branch or diff for regressions, missing tests, migration risks, and edge cases, including branch auto-commenting rules and local review workflows.
- [x] Bug triage and fixes: tracing rendering or config bugs, identifying the code path, patching the issue, and running targeted tests, including closing GitHub issues when appropriate.
- [x] Test generation: adding or repairing unit tests and Storybook tests for visualization and editor behavior.
- [x] Migration work: updating shared config migrations, version handling, and legacy compatibility logic after checking the migration docs first.
- [x] Config maintenance: updating the relevant `CONFIG.md` docs whenever consumer-facing config changes.
- [x] Refactors: renaming APIs, moving shared helpers, tightening package boundaries, and cleaning up deprecated paths.
- [x] Repo documentation: maintaining READMEs, developer docs, and internal context docs for complex systems.
- [x] Branch analysis: comparing a feature branch against `dev` and summarizing what changed.
- [x] Test execution workflows: running Playwright, Backstop, Jest, and related test tooling.

## GOALS. We are working towards...
- [ ] QA Team Onboarding
- [ ] Automatic Fixing/Closing Issues from Github/Dashboards
- [ ] Automatic Accessibility Reviews in Storybook
